# Структура проекта Encontro

## Основные компоненты

### 1. Backend (Go)
```
├── cmd/                    # Точки входа в приложение
├── internal/              # Внутренняя логика приложения
│   ├── domain/           # Бизнес-модели и интерфейсы
│   ├── usecase/          # Бизнес-логика
│   ├── delivery/         # Обработчики HTTP/WebSocket
│   └── infrastructure/   # Реализации репозиториев и внешних сервисов
├── certs/                # SSL сертификаты
├── static/              # Статические файлы для сервера
└── scripts/             # Вспомогательные скрипты
```

### 2. Frontend (TypeScript/Vue.js)
```
├── frontend/
│   ├── src/             # Исходный код
│   ├── public/          # Публичные статические файлы
│   ├── .vite/           # Кэш сборки
│   └── node_modules/    # Зависимости
```

### 3. Вспомогательные компоненты
```
├── mockServer/          # Мок-сервер для тестирования
├── .vscode/            # Настройки VS Code
└── scripts/            # Скрипты для разработки и деплоя
```

## Принцип работы

### Backend (Go)
1. **Архитектура**: Проект следует чистой архитектуре (Clean Architecture)
   - `domain/`: Содержит бизнес-модели и интерфейсы
   - `usecase/`: Реализует бизнес-логику
   - `delivery/`: Обрабатывает HTTP/WebSocket запросы
   - `infrastructure/`: Содержит реализации для работы с БД и внешними сервисами

2. **Основные компоненты**:
   - Signaling сервер для WebRTC
   - HTTP API для управления комнатами и пользователями
   - WebSocket для real-time коммуникации

### Frontend (Vue.js)
1. **Технологии**:
   - Vue.js 3 с Composition API
   - TypeScript для типизации
   - Vite как сборщик
   - WebRTC для peer-to-peer коммуникации

2. **Структура**:
   - Компонентный подход
   - Модульная архитектура
   - Интеграция с backend через REST API и WebSocket

## Как добавить новый функционал

1. **Backend**:
   - Добавить новые модели в `internal/domain/`
   - Реализовать бизнес-логику в `internal/usecase/`
   - Создать обработчики в `internal/delivery/`
   - При необходимости добавить новые репозитории в `internal/infrastructure/`

2. **Frontend**:
   - Создать новые компоненты в `frontend/src/components/`
   - Добавить новые страницы в `frontend/src/views/`
   - Обновить типы в `frontend/src/types/`
   - При необходимости добавить новые API методы в `frontend/src/api/`

## Как обновить зависимости

1. **Backend**:
   ```bash
   go get -u ./...  # Обновить все зависимости
   go mod tidy      # Очистить неиспользуемые зависимости
   ```

2. **Frontend**:
   ```bash
   cd frontend
   npm update       # Обновить зависимости до последних совместимых версий
   npm audit fix    # Исправить уязвимости
   ```

## Как удалить неиспользуемый код

1. Использовать инструменты статического анализа:
   - `go vet` для Go кода
   - ESLint для TypeScript/JavaScript
2. Проверить неиспользуемые импорты
3. Удалить неиспользуемые компоненты и файлы
4. Обновить зависимости после удаления кода

## Важные файлы конфигурации

1. **Backend**:
   - `go.mod` и `go.sum` - управление зависимостями
   - `.gitignore` - исключения из системы контроля версий

2. **Frontend**:
   - `package.json` - зависимости и скрипты
   - `vite.config.ts` - конфигурация сборки
   - `tsconfig.json` - настройки TypeScript
   - `.prettierrc.json` - форматирование кода

## Дополнительная информация

### Разработка
- Для локальной разработки используйте `go run cmd/server/main.go` для запуска backend
- Frontend запускается через `cd frontend && npm run dev`
- Для тестирования используйте `go test ./...` и `npm test` соответственно

### Деплой
- Backend собирается в Docker контейнер
- Frontend собирается в статические файлы через `npm run build`
- Все конфигурации окружения хранятся в `.env` файлах (не включены в репозиторий)

### Безопасность
- Все чувствительные данные хранятся в переменных окружения
- SSL сертификаты находятся в директории `certs/`
- API ключи и токены не должны быть закоммичены в репозиторий

### Мониторинг
- Логирование реализовано через стандартный пакет `log`
- Метрики можно собирать через Prometheus (конфигурация в `internal/infrastructure/metrics/`)
- Трейсинг реализован через OpenTelemetry

### Тестирование
- Unit тесты находятся рядом с тестируемым кодом
- Интеграционные тесты в `internal/tests/`
- E2E тесты в `frontend/tests/e2e/`
- Моки для тестирования в `mockServer/`
